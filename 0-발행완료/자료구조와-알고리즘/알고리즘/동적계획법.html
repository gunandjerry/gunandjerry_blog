동적계획법

더 작은 계산의 결과를 저장해놓고(Memoization) 이를 보다 큰 계산에 활용하여 최종적인 결과값을 찾아내는 방법론.
'가장 큰 값'부터 아래로 내려가며 캐싱 배열을 채우면 탑-다운 방식, 반대로 '가장 작은 값'부터 위로 올라가며 채우면 바텀-업 방식이라고 부른다.

가장 대표적인 예시는 피보나치 수열 구하기로, 탑-다운 방식과 바텀-업 방식으로 구현하면 다음과 같다.

// 탑-다운 방식.
// 큰 수부터 재귀적으로 내려가며 memo에 값을 기록하고, 이미 기록된 값을 다시 사용한다면 바로 반환하는 방식의 최적화
int memo[1000]; memset(memo, -1, sizeof(memo));
int fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = fib(n-1) + fib(n-2);
}

// 바텀-업 방식.
// 작은 수부터 값을 기록해가며 최종적인 목표값을 찾는다.
int fib(int n) {
    vector<int> dp(n+1);
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}

이 예시에서는 N번째 피보나치 수를 구하기 위해선 그 전의 피보나치 수들을 모두 알아야 하기 때문에 불필요한 재귀 호출 비용이 없고 캐시 효율이 좋은 바텀 업 방식이 더 빠르지만(실제 시간 측정시. 빅오 표기론 동일하게 O(N)이다), 상황에 따라 굳이 모든 경우를 채워가며 memo 배열을 만들 필요가 없을 땐 탑-다운이 더 효율적이기도 하다.

배낭 문제 (Knapsack problem)

DP로 풀 수 있는 가장 전형적인 조합 최적화 문제. 최대 K만큼의 무게를 담을 수 있는 가방이 있고, 물품마다 각각 무게와 가치가 매겨질 때, 가방에 담은 물건들의 가치 합이 최대가 되도록 담는 방법을 구한다.

핵심은 가지치기에 있다. '넣는다'와 '안 넣는다'의 2지선다로 완전탐색을 하는 경우 경우의 수는 이 되는데, 이 때 '이 선택을 해봐야 반드시 손해'임을 해보지 않아도 알 수 있다면 전부 제거해버리는 것.

냅색 문제에서는 이 가지치기가 '그 때까지 최선의 수만을 남겨놓는' 방식으로 구현된다. 즉, 최선의 수가 아닌 모든 경우의 수를 제거함으로써 훨씬 빠르게 문제를 해결할 수 있게 된다.

그렇다면 어떻게 최선의 수만 남길 수 있을까? 간단하다.

  1. dp 배열을 만들고 dp[i]가 무엇을 의미하는지 정의한다. 여기선 가방에 i 무게 만큼의 물건을 담았을 때, 가능한 최대 가치로 정의한다.
  2. 각각의 물건에 대해 dp 배열을 순회하며 이 물건을 담았을 때 가치의 총량이 얼마나 되는지 체크하고, 기존에 기록되어 있던 가치의 합보다 크다면 갱신한다.

이를 점화식으로 정리해보자. 현재 물건의 무게가 w, 가치가 v라고 할 때
dp[i] > dp[j-w] + v 이면 dp[i]를 dp[j-w] + v로 갱신한다.

2차원 배열 풀이

앞서 이미 1차원 배열만으로 풀 수 있음을 암시했지만, 가장 기본적인 형태는 2차원 배열로 풀이하는 것이다.

// N개의 물품 각각에 가치와 무게가 있고, 배낭이 담을 수 있는 최대 무게가 K일 때, 가치의 최대가 되도록 담는 법
vector<int> weights{ ... };
vector<int> values{ ... };

// dp[i][j]는 i번째 아이템까지 확인했을 때, 배낭에 담긴 물건의 무게가 총 j일 때, 가치의 최대
vector<vector<int>> dp(max_weight + 1, vector<int>(max_weight + 1, 0));
for (int i = 1; i <= n; i++) // 아이템 1 ~ i번 품목까지 검토
{
    for (int j = 1; j <= max_weight; j++) // 무게 j까지 허용
    {
		    // 아이템을 담으려면 현재 한계 무게(j)가 현재 아이템의 무게 이상이여서 j-w가 0 이상이어야 한다.
        if (weights[i] <= j)
        {
		        // 담지 않는 선택과 담는 선택 중 최선의 수를 고른다
            dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - weights[i]] + values[i]);
        }
        // 현재 아이템을 담기에는 현재 한계 무게(j)가 너무 작다면 이전 값을 그대로 상속한다.
        else
        {
            dp[i][j] = dp[i - 1][j];
        }
    }
}

cout << dp[n][max_weight];

1차원 배열 풀이

간단한 배낭 문제는 어차피 모든 물품을 순회하며 현재 한계 무게가 1 ~ max_weight 일 때를 모두 체크하므로,
두 개의 1차원 배열만 있으면 된다.

vector<int> weights{ ... };
vector<int> values{ ... };

vector<int> dp(max_weight + 1, 0);
vector<int> backup(max_weight + 1, 0);
for (int i = 1; i <= n; i++) // 아이템 1 ~ i번 품목까지 검토
{
    for (int j = weights[i]; j <= max_weight; j++) // 무게 j까지 허용
    {
        dp[j] = max({ dp[j], backup[j - weights[i]] + values[i] });
    }
    backup = dp;
}

cout << dp[max_weight];

합이 최대가 되는 연속 구간 구하기

Kadane’s Algorithm.
음수 값이 있는 배열에서 구간 합이 최대가 되는 구간의 합을 구하는 알고리즘.

dp[i]는 i번째 요소까지 확인했을 때 가장 높은 구간합을 기록하고, i+1 번째 요소와 dp[i]를 비교해 더 큰 값으로 dp[i+1]을 갱신하기만 하면 된다.

왜냐면 구간합은 어디까지나 더하기 연산으로 이루어지므로, 이전 요소까지 확인했을 때 최선의 값이 현재 요소 하나보다도 못 하다면 그냥 버리고 다시 시작하는게 반드시 더 낫기 때문이다.

vector<int> nums{ ... };  // n개
vector<int> dp(n, 0);

int cur_max = nums[0];
for (int i = 1; i < n; ++i)
{
		dp[i] = max(nums[i], dp[i - 1] + nums[i]);
		cur_max = max(cur_max, dp[i]);
}

cout << cur_max;

연속으로 N개만 고를 수 있을 때 최대로 이득보기

"밟으면 거기에 써있는 점수를 얻는 계단이 있다(음의 점수는 없다). 연속된 계단은 최대 2번까지만 밟을 수 있고, 그 이후로는 한 개의 계단을 건너뛰어야만 한다. 계단의 정상에 도달했을 때 얻을 수 있는 최대 점수를 구하라."

매우 자주 나오는 유형이다. 풀이법은 배낭 문제와 크게 다르지 않다.

  • dp[i]는 i번 계단을 밟았을 때 얻을 수 있는 최대 점수라고 정의한다.
  • 만약 N번째 계단을 밟는다면, (1) N-2번 계단을 밟은 뒤 하나를 건너 뛰고 밟았거나, (2) N-2번 계단을 건너뛰고 N-1번 계단을 밟은 뒤 밟았거나, 이 둘 중 하나이다.

그렇다면 점화식을 세워보자.

  • 2번 전을 취하고 이번 것을 취함: dp[i-2] + value[i]
  • 3번 전 것, 1번 전 것, 이번 것을 취함: dp[i-3] + value[i-1] + value[i]

이제 둘 중 더 높은 점수를 기록하기만 하면 된다. 점화식에서 dp[i-1]은 사용하지 않기 때문에 dp[i]에 값을 담을 때 i번째 계단을 '밟지 않는다'는 선택은 고려하지 않아도 된다. 어차피 그 다음 계단에서의 값을 구할 땐 dp[i]를 사용하지 않고 i번째 계단을 밟지 않은 경우를 알아서 고려하며, 맨 마지막 계단은 반드시 밟아야 하기 때문이다.

vector<int> value{ ... }; // n개
vector<int> dp(n);

// dp의 첫 세 요소는 수동으로 초기화한다. 점화식에서 최대 i-3까지 접근하기 때문.
dp[0] = value[0];
dp[1] = value[0] + value[1]; // 두 번째 계단의 최대점수는 반드시 두 개를 모두 밟았을 때
dp[2] = max(value[0] + value[2], value[1] + value[2]); // 세 번째 계단의 최대점수는 두 개의 경우의 수가 있음.

for (int i = 3; i < n; ++i)
{
		dp[i] = dp[i - 2] + value[i]; // 2번 전과 이번 것을 취했을 때
		dp[i] = max(dp[i], dp[i - 3] + value[i - 1] + value[i]); // 3번 전, 1번 전, 이번 것을 취했을 때
}

2차원 배열로 풀기

위의 풀이가 보다 정석적이지만, 2차원 배열로 풀 수도 있다.
dp[i][0]은 i번째 계단을 '한 칸 올라서' 밟은 경우고, dp[i][1]은 i번째 계단을 '두 칸 올라서' 밟은 경우라고 해보자.

vector<int> value{ ... }; // n개
vector<int> dp(n);

dp[0][0] = value[0]; // 첫 번째 계단은 한 칸 올라서 밟는 경우밖에 없다
dp[1][0] = value[0] + value[1]; // 두 번째 계단을 한 칸 올라서 밟을 때의 최대
dp[1][1] = value[1]; // 두 번째 계단을 두 칸 올라서 밟을 때(즉, 첫 번째 계단을 생략할 때)의 최대

for (int i = 2; i < n; ++i)
{
		// 전의 계단에서 한 칸 올라 밟는 경우. 그러려면 전의 계단은 2칸 올라와서 밟았어야 한다.
		dp[i][0] = dp[i - 1][1] + value[i];
		// 전의 계단에서 두 칸 올라 밟는 경우. 이 경우엔 2칸 전의 계단을 어떻게 밟았는 지는 상관 없다.
		dp[i][1] = max(dp[i - 2][0], dp[i - 2][1]) + value[i];
}

가장 긴 증가하는 부분 수열(LIS)의 길이

{ 1, 2, 1, 3, 2, 5 }에서 LIS는 { 1, 2, 3, 5 } 이다.

1. 원본 순서를 유지할 때

첫 번째 풀이는 으로 수열의 원본 순서를 유지하며 확인하는 방법이다. 만약 LIS의 길이만 구하는 것이 아닌 그렇게 얻어진 LIS를 반환해야 한다면 이 방식을 써야 한다.

원리는 완전 탐색에 심플한 가지치기를 입힌 것이다.

  1. 이중 for문으로 두 개의 포인터를 둔다. 외곽 for문의 포인터 i는 1 ~ n-1 인덱스의 요소를 차례로 가리킨다. 내부 for문의 포인터 j는 0 ~ i-1 인덱스의 요소를 차례로 가리킨다.
  2. i가 가리키는 것이 기준 요소가 된다. j를 0 ~ i-1로 옮겨가며 j가 가리키는 값이 i가 가리키는 값보다 작다면 i는 현재 j가 가리키고 있는 위치까지 만들어진 증가하는 부분 수열에 추가할 수 있다는 의미가 된다.
  3. 따라서 dp[i]는 dp[j] + 1이다.

이 때 dp[j]에도 그 때까지의 최고 길이가 저장되어 있으므로 dp[j] + 1을 하는 것만으로도 계속해서 최고 길이를 갱신해갈 수 있게 된다.

vector<int> nums{ ... }; // n개
vector<int> dp(n, 1); // 자기 혼자만으로도 길이 1의 수열이 되므로, 초기값은 1

int max_len = 1; // LIS 길이
for (int i = 1; i < n; ++i)
{
		for (int j = 0; j < i; ++j)
		{
				// 어떤 j에서 이미 최고길이를 dp[i]에 갱신해놓고 그 다음 j에서 더 짧은 길이로 덮어쓰지 않도록 dp[j] >= dp[i] 조건도 만족해야 함에 주의
				if (nums[i] > nums[j] && dp[j] >= dp[i])
				{
						dp[i] = dp[j] + 1;
						if (dp[i] > max_len) max_len = dp[i];
				}
		}
}

2. 원본 순서를 유지하지 않아도 된다면

길이만 구한다면 이진탐색을 써서 훨씬 빠르게 풀 수 있다. 시간복잡도는

원리는 이렇다.

  1. L 배열을 하나 둔다. L[x]는 길이가 x인 증가하는 수열의 마지막 요소의 값이다. 핵심은 L[x]가 항상 '최소가 되도록' 유지하는 것에 있다.
  2. 예컨대 { 3, 5, 1, 2, 4 }이란 수열이 있다고 해보자. 첫 번째 요소부터 순회한다. L[1]은 3이다.
  3. 다음 5를 보자. 길이가 1인 증가하는 수열의 마지막 요소(L[1])가 3인데, 그것보다 크므로 3 -> 5란 수열을 만들 수 있다. 따라서 L[2]는 5이다.
  4. 다음 1을 보자. 이 수는 L[2] 보다 작고, L[1]보다도 작다. 따라서 다른 수열에 추가될 수 없다. 하지만 그냥 버리지는 않는다. 생각해보자. 현재 L[1]은 3이고, 이는 '길이가 1인 증가하는 수열'에 다음 수가 붙으려면 3보다 커야 한다는 뜻이다. 그렇다면 L[1]의 값을 지금 수인 1로 바꾸는 쪽이 더 이득이 되지 않을까? 따라서 L[1]을 1로 갱신한다.
  5. 다음 2를 보자. 만약 4번에서 L[1]을 1로 바꾸지 않았다면, 2는 3보다 작으므로 붙을 수가 없었다. 그런데 4번에서 L[1]을 1로 갱신해놨기 때문에, 우리는 1 -> 2로 수열을 만들 수 있다! 그렇다면 또 생각해보자. 현재 L[2]는 5이다. 하지만 길이가 2인 증가하는 수열의 마지막 값이 5인 것보다 현재 수인 2인 것이 더 유리하다. 따라서 L[2]를 2로 갱신한다.
  6. 마지막으로 4를 보자. 이 수는 현재 L[2]인 2보다 크기 때문에, 바로 붙을 수 있다. 따라서 L[3]는 4이다.

이렇게 모든 요소를 순회했다면 L배열의 길이(위 예시에선 3)가 곧 LIS의 길이가 된다.
주의해야 할 것은 이 과정에서 L에 기록되는 값들은 어디까지나 '길이가 x인 증가수열의 마지막에 올 수 있는 가장 낮은 수'이지 실제 LIS를 기록해둔 것은 아니라는 것이다.
예컨대 { 2, 3, 1, 6, 4, 5 } 배열에 대해 L 배열을 만들어보면 { 1, 3, 4, 5 }가 나오는데, 실제 LIS는 { 2, 3, 4, 5 }로 일치하지 않는다. L 배열의 1은 어디까지나 '길이가 1인 증가하는 부분 수열로 올 수 있는 가장 낮은(가장 유리한) 값은 1임'을 의미한다. 즉, 현재 길이가 1인 증가하는 수열에 값을 하나 더 붙이려고 할 때, 2 -> 로 시작하는 것보다 1 -> 로 시작해서 만드는게 유리하다는 의미다.

여기서 시간복잡도를 낮추는 핵심은 L배열이 오름차순으로 정렬되기 때문에 '현재 수 X 이상인 첫 번째 요소가 L배열의 몇 번째 인덱스에 있느냐'를 이진 탐색으로 찾을 수 있다는 데 있다.

dp 배열은?

x번째 요소까지 확인했을 때 LIS의 길이를 뜻하는 dp[x] 배열도 여전히 만들 수 있다. (필요하다면)
L[i]의 i가 곧 길이가 i인 수열을 뜻하므로, dp[x]에 i를 넣으면 끝이다. 단, 인덱스가 0부터 시작하므로 + 1.

vector<int> nums{ ... }; // n개
vector<int> L;
// vector<int> dp; // 필요하면

for (int i = 0; i < n; ++i)
{
		auto iter = lower_bound(L.begin(), L.end(), nums[i]);
		// dp[i] = iter - L.begin() + 1; // 필요하면
		
		// 현재 L 배열에 이 값 이상이 없다면 -> LIS의 길이가 1 증가
		if (iter == dp.end())
		{
				L.push_back(nums[i]);
		}
		// 있다면 더 유리한 수로 갱신. lower_bound로 찾았으므로 iter가 가리키는 L 배열의 기존 값은 항상 nums[i] 이상이므로, 반드시 현재 값 이하의 값으로 갱신된다.
		else
		{
				*iter = nums[i];
		}
}

return L.size();

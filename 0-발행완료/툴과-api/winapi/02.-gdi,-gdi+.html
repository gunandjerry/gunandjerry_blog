2. GDI, GDI+

Graphics Device Interface

WinAPI를 위한 그래픽 API. 현재는 Direct2D와 DirectWrite에 의해 완전히 대체되었다.
일부 기능에서 하드웨어 가속을 지원하긴 하지만, 이후 나온 API들과 비교하기 미안할 정도로 느리다.

GDI+는 WindowsXP 시절에 나온 업그레이드판.

창 다시 그리기

기본적으로 WinAPI 어플리케이션은 다시 그려야 할 이유가 생기지 않으면 WM_PAINT 메시지를 발생시키지 않는다. 하지만 게임은 틱마다 화면을 다시 그려야 하므로, 다음과 같은 방법을 사용한다.

InvalidateRect(hWnd, nullptr, false); // 윈도우 전체 영역이 다시 그려져야 한다고 설정한다.

GDI 시작하기

Device Context (DC)

모든 창은 자신의 DC를 가지며 이 DC에는 여러가지 그래픽 정보가 들어있다. GDI는 창의 DC 핸들을 가져와 DC에 그리기 데이터를 주입함으로써 창에 그림을 그린다.

사용 방법은 다음과 같다.

  1. 창의 DC를 얻어와 연결한다.
  2. Create@@@ 류 함수로 GDI 객체를 생성한다.
  3. 창의 DC의 GDI 객체를 새로 생성한 것으로 교체한다.
  4. 그린다.
  5. 그리기 작업이 끝나면 DC는 원래대로 복구하고 2번에서 생성한 GDI 객체는 제거한다.
HDC hdc = GetDC(hWnd);
// 임의로 생성한 GDI 브러시 객체. 브러시 객체는 도형 내부의 색을 채우는 역할
HBRUSH myBrush = CreateSolidBrush(RGB(255, 0, 0));

// 임의로 생성한 GDI 객체는 사용이 끝나면 제거하고 DC의 원본 데이터를 복구해야 한다.
// 따라서 DC의 원래 Brush 객체를 담을 핸들도 필요하다.
HBRUSH originalBrush = nullptr;

// SelectObject() 함수는 DC의 GDI 객체를 교체하는데, 이 때 원본 GDI 객체의 핸들도 반환해준다.
// 이 때 매개변수로 전달되는 타입에 따라 알맞은 DC의 GDI 객체를 교체해주므로 호출할 때는 신경쓸 필요가 없지만, 반환되는 원본 GDI 객체의 핸들은 모두 HGDIOBJ 타입(모든 GDI 객체 핸들의 부모 타입)으로 반환되기 때문에 저장하려면 형변환을 해줘야 한다.
originalBrush = reinterpret_cast<HBRUSH>(SelectObject(hdc, myBrush));

// myBrush를 가진 상태로 그리기
Rectangle(hdc, 10, 10, 110, 110);

// 그리기 작업이 끝나면 DC를 원래대로 복구하여 예기치 못 한 사태를 방지하고, 임의로 생성한 GDI 객체는 메모리 자원을 차지하므로 제거해주자.
// 참고로 GDI 객체는 Select된 상태에서는 제거할 수 없기 때문에 임의로 생성한 GDI 객체를 제거하려면 어차피 다른 GDI 객체로 교체해줘야 한다.
SelectObject(hdc, originalBrush);
DeleteObject(myBrush);

ReleaseDC(hWnd, hdc);

자주 사용되는 GDI 객체들은 다음과 같다.

// HBRUSH - 도형의 내부를 채운다.
// CreateSolidBrush - 단색
HBRUSH myBrush = CreateSolidBrush(RGB(255, 0, 0)); // 빨간색 단색으로 채운다.
// CreateHatchBrush - 빗금
myBrush = CreateHatchBrush(HS_BDIAGONAL, RGB(255, 0, 0)); // 빨간 대각선으로 채운다.
// Hatch의 종류
// HS_HORIZONTAL  ------
// HS_VERTICAL    |||||
// HS_FDIAGONAL   \\\\\
// HS_BDIAGONAL   /////
// HS_CROSS       +++++
// HS_DIAGCROSS   xxxxx


// HPEN - 선, 도형의 윤곽선
HPEN myPen = CreatePen(PS_SOLID, 5, RGB(255, 0, 0)); // 굵기 5, 빨간색 실선
// Pen Style 종류
// PS_SOLID  실선
// PS_DASH   ----
// PS_DOT    점선
// PS_NULL   선(윤곽선)을 그리지 않을 때
// 등등... Dash나 점선같은 것들은 굵기가 1이 아니면 안 된다.


// HFONT - 문자의 폰트
HFONT myFont = CreateFont(20, 0, 0, 0, FW_NORMAL, 0, 0, 0, ANSI_CHARSET, 0, 0, 0, 0 TEXT("Consolas"));
// 1번 인수: 문자의 높이. 2번 인수는 문자의 너비인데 0을 전달하면 알아서 1번 인수에 맞춰서 설정되므로 사실상 1번 인수가 글자의 크기가 된다.
// 5번 인수: 글꼴의 굵기. 0 ~ 1000. 기본 값은 FW_NORMAL(400)
// 9번 인수: 문자 집합. ANSI_CHARSET, OEM_CHARSET, HANGEUL_CHARSET 등이 있다.
// 14번 인수: 사용할 폰트명. 당연히 윈도우 fonts 폴더에 있는 글꼴이어야 한다.


// 가장 중요한 GDI 객체 중 하나인 비트맵 객체에 대해선 나중에 배우자.

StockObject

WinAPI가 미리 만들어둔 GDI 객체를 StockObject라고 한다. 다음과 같이 사용하며, 기본적으로 만들어져 있는 놈들이므로 사용이 끝나면 DC만 복구하고 StockObject는 제거하지 않는다.

myBrush = reinterpret_cast<HBRUSH>(GetStockObject(NULL_BRUSH)); // 도형 내부를 투명하게 채울 때 NULL_BRUSH

GDI+

GDI 라이브러리의 확장판이다. 상당히 많은 기능들이 추가되었다. 대표적인 변화는 다음과 같다.

  1. 2D 벡터 그래픽을 지원한다.
  2. 비트맵 등을 출력할 때 자주 사용되는 이미지를 캐싱하여 속도가 빨라졌다.
  3. 텍스트, 그리기에 안티앨리어싱을 지원한다.

Windows.h를 포함하면 알아서 포함되던 GDI와 달리 GDI+를 사용하기 위해선 여러 가지 추가 작업들이 필요하다.

  1. 필요한 헤더와 라이브러리를 포함한다.
#include <objidl.h>
#include <gdiplus.h>
#pragma comment (lib,"Gdiplus.lib")
  1. 사용하기 전에 초기화하고, 프로그램을 종료하기 전에 제거해야 한다.
Gdiplus::GdiplusStartupInput gdiplusStartupInput;
ULONG_PTR gdiplusToken;
Gdiplus::GdiplusStartup(&gdiplusToken, &gdiplusStartupInput, NULL);

...(메인 메시지 루프)

Gdiplus::GdiplusShutdown(gdiplusToken);

모든 GDI+ 객체나 함수들은 Gdiplus 네임스페이스에 정의되어 있다. using namespace를 해도 되지만 그럼 GDI와 모호한 이름 문제가 발생할 수도 있어 웬만하면 네임스페이스를 써주는 걸 추천한다.

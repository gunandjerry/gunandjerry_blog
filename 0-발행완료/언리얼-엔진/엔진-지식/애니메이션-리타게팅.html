애니메이션 리타게팅

개념

서로 다른 본을 가진 스켈레톤 간에 하나의 애니메이션 시퀀스를 공유하여 사용하기 위한 기능.

예를 들어 표준적인 체형의 캐릭터를 기준으로 만들어진 스켈레톤과 그 스켈레톤을 기준으로 만들어진 애니메이션이 있다고 했을 때, 생김새가 다른 스켈레톤을 사용하는 캐릭터가 해당 애니메이션을 사용하려고 하면 아래와 같이 골격의 차이로 인한 문제가 발생하게 된다.

위 그림에서는 파츠가 조각 난 것으로 표현했지만 실제로는 스키닝된 캐릭터를 사용할 것이므로 관절이 이상한 모양이 되거나, 특정 위치의 피부가 늘어나거나, 어깨가 좁아지는 등의 문제로 나타날 것이다.

따라서 애니메이션을 이루는 키 프레임의 값들을 다른 형태의 스켈레톤에 맞게 조정해줘야 하는데, 이 작업을 애니메이션 리타게팅이라고 한다.

위 그림은 애니메이션 리타게팅을 통해 골격이 다른 캐릭터에 같은 애니메이션을 적용한 결과이다.
리타게팅으로 변하는 건 본의 Translation 값 뿐이며 Rotation 값은 언제나 애니메이션의 키 프레임 값을 따르게 된다.

Note

참고로 유니티의 경우 휴머노이드 캐릭터라는 보편적인 모양의 스켈레톤을 기준으로 삼고 리타게팅 한다.

리타게팅 수행하기

Bone Translation Retargeting Mode

리타게팅은 각각의 본에 대해 수행한다. 스켈레톤 에디터의 Skeleton Tree 섹션에서 우상단 톱니바퀴 -> Show Retargeting Options 옵션을 활성화한다.

이제 각각의 본에 대해 Bone Translation Retargeting Mode를 설정할 수 있다.

모드는 다음 세 가지만 알면 충분하다.

Animation

애니메이션의 키 프레임에 있는 본의 Translation(위치) 값을 그대로 사용한다. 즉, 해당 본에 대해 리타게팅을 수행하지 않는다.

Skeleton

애니메이션의 키 프레임에 있는 본의 Translation(위치) 값을 완전히 무시하고, 자기 스켈레톤의 바인드 포즈의 뼈 위치(상대적인)만을 사용한다.

Animation Scaled

애니메이션의 키 프레임에 있는 본의 Translation(위치) 값을 가져오되, 애니메이션의 스켈레톤과 캐릭터의 스켈레톤의 본 길이 비율을 계산해 그 비율만큼 스케일링을 수행한다.

본이 너무 많아서 곤란하다면 상위 본을 우클릭하고 Recursively Set 메뉴를 선택하면 모든 하위 본에 일괄 설정된다.

본 이름이 같은 스켈레톤 간의 리타게팅

기본적으로 애니메이션의 적용은 본의 이름을 통해 이루어지므로, 본의 이름은 같고 각 본의 Translation에만 차이가 있는 두 스켈레톤 간에는 별도의 처리가 필요하지 않다.
말인 즉, Bone Translation Retargeting Mode만 설정해주고 애님 인스턴스를 그대로 사용하면 된다.

같은 스켈레톤 간의 리타게팅(?)

캐릭터의 스켈레탈 메쉬와 애니메이션이 하나의 스켈레톤을 공유하더라도 Translation Retargeting Mode를 설정해줄 필요가 있을 수도 있다. 다음 문서를 참고하자. (근데 이런 경우도 리타게팅이라고 부르는 지는 모르겠다.)

일반적으로 애니메이션은 기본 스켈레톤 기반의 회전 데이터로만 구성되므로 Animation이나 Animation Scaled를 써야 하는 경우는 많지 않다. 전형적인 설정은 다음과 같다.

  1. 루트 본, IK 본, 웨폰 본이나 마커같이 애니메이션이 움직일 때 같이 움직여야 하는 본은 Animation 모드를 사용한다.
  2. 캐릭터의 중심이 되는 본(기본은 Pelvis)은 AnimationScaled 모드를 사용한다. 달리 애니메이션의 Translation을 반영해야 하는 본도 마찬가지. 이걸 해줘야 상하체의 위치를 옮기고 앉거나 할 수 있다.
  3. 다른 본들은 Skeleton 모드를 사용한다.

본 이름이 다른 스켈레톤 간의 리타게팅

이름이 다른 스켈레톤은 서로 호환되지 않으므로 같은 부위의 본들 또는 본들의 묶음에 같은 별칭을 붙여 서로 매핑해야 한다. IK Rig 에셋으로 처리한다.

컨텐츠 브라우저 우클릭 -> Animation -> Retargeting -> IK Rig 에셋을 생성하고 에디터를 연다.

Preview Skeletal Mesh에 스켈레탈 메쉬를 부착하면 계층 창에 본 계층이 표시된다.

가장 먼저 리타겟 루트를 설정한다. 일반적으로 캐릭터의 중심 본인 Pelvis를 사용하며, 언리얼 5.6에선 본을 우클릭하고 Set Pelvis를 선택하면 된다.

그 다음 Start Bone ~ End Bone을 하나로 묶어 리타겟 체인을 만든다. 직계 관계의 여러 본을 선택하고 New Retarget Chain 옵션을 선택한다.

IK Retargeting 탭에서 생성된 리타겟 체인을 확인할 수 있다.

이런 과정을 모든 본에 대해 두 스켈레탈 메쉬에 대해 진행해주면 된다.

IK Rig 에셋의 구성이 끝났다면 두 IK Rig를 연결할 IK Retargeter 에셋을 생성한다. Create Asset 메뉴의 같은 위치에 있다.

Source, Target IK Rig를 선택하고 미리보기용 메쉬를 부착하자.
뷰포트 상에서 두 메쉬가 겹치면 Mesh Offset을 조정하면 된다.

Chain Mapping 탭에서 IK Rig에서 설정한 Retarget Chain들을 각각 매핑한다.

매핑이 완료되었다면 Asset Browser에서 애니메이션 시퀀스를 눌러 두 스켈레탈 메쉬가 잘 동기화되었는지 확인할 수 있다.

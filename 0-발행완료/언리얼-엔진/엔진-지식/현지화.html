현지화

사전 작업

쿠킹, 패키징에 포함할 언어 설정

Project Settings - Packaging - Packaging - Advanced - Languages to Package에 지원할 언어를 추가한다.

국제화 지원 세트 변경

언리얼은 ICU 국제화 라이브러리를 사용하며, 이를 통해 OS의 언어를 파악하거나 유니코드 호환 변환(ToUpper같은), 비교와 정렬 등을 수행한다고 한다. 자세한 건 공식문서(문서)를 참고하자.
Project Settings - Packaging - Packaging - Advanced - Internationalization Support의 설정을 앞서 Languages to Package에서 추가했던 모든 언어를 포괄할 수 있는 것으로 설정한다. (웬만하면 EFIGSCJK로 충분하다.)

이걸 설정을 안 하면 기껏 현지화를 위한 작업을 해놔도 패키징 할 땐 그 데이터들이 다 빠지거나 문제가 생길 수 있다고 한다.

텍스트 현지화

Localization Dashboard 사용하기

Tools - Localization Dashboard에서 텍스트 현지화를 총괄한다.
현지화가 필요한 텍스트 데이터의 수와 현지화 현황을 파악할 수 있다.

전반적인 현지화 과정은 다음과 같다.

  1. 새 현지화 대상(Target)을 생성한다.
  2. 텍스트 데이터를 수집한다.
  3. 텍스트 데이터를 .po 파일로 내보낸다.
  4. poedit 프로그램으로 번역한 뒤 그 .po 파일을다시 임포트한다.

현지화 대상은 기본 게임 타겟 하나면 충분하다.

텍스트 데이터 수집하기

Gather from Packages에 체크하고 텍스트 데이터로 수집할 에셋들이 있는 경로를 입력한다.

이후 하단의 Gather Text 버튼을 누르면 설정된 경로에 있는 모든 현지화 가능한 텍스트 데이터가 수집된다. 해당 텍스트 데이터들은 기본 언어로 작성되어 있을 것이다.

하단의 Add New Language를 눌러 지원할 언어를 추가한다.

poedit으로 번역하기

개별 언어 우측의 또는 상단의 커다란 Export Text 버튼을 눌러 .po 파일로 내보낸다.
개별 언어에 대해 수행했다면 해당 언어만, 상단의 큰 버튼을 통하면 언어별로 폴더로 정리되어 내보내진다.

이후 poedit에서 .po 파일을 열어 번역한다.

번역한 파일 다시 불러오기

Localization Dashboard로 돌아가 개별 언어 우측의 또는 상단의 Import Text 버튼을 눌러 번역이 완료된 .po 파일을 불러온다. 상단의 큰 버튼을 이용하면 내보낼 때와 마찬가지로 상위 폴더를 지정하여 아래에 있는 모든 .po 파일을 불러올 수 있다.

Localization Dashboard에서 현재 어떤 언어가 얼만큼 번역이 진행되었는지 추적할 수 있다.

작업을 마쳤다면 프로젝트/Localization/Game/ 경로 아래에 관련 파일이 알아서 생성되며, 패키징을 하면 알아서 알맞은 언어를 사용한다.

FText 사용하기

FText 타입을 사용하는 것만으로 쉽게 현지화 시스템과 연결될 수 있다. 블루프린트에서 FText는 현재 프로젝트에 존재하는 모든 String Table의 데이터 목록 중 특정 키를 고를 수 있다.

String Table 에셋은 대표적인 현지화 가능한 텍스트 데이터 에셋으로, Localization Dashboard에서 Gather Text를 하면 모든 행의 데이터가 수집되므로 별도의 절차 없이 현지화를 적용할 수 있다.

폰트 에셋

언리얼의 Composite Font(복합 폰트) 에셋은 아주 쉽게 문화별로 폰트를 지정할 수 있게 해준다.
Font 에셋의 구성은 다음과 같다.

Default Font Family

기본 폰트를 설정한다.

Fallback Font Family

기본 폰트가 커버하지 못 하는 문자가 있다면 대안으로 사용이 검토된다.

Sub-Font Family

유니코드 범위별로 특정 폰트를 사용하도록 한다. 사용 조건에 부합되는 경우 기본 폰트보다 우선순위가 높게 사용된다.

문화(언어)별로 다른 폰트 사용하기

특히 Composite Font 에셋의 Sub-Font Family를 사용하면 쉽게 문화별로 다른 폰트를 사용하게 할 수 있다.
예컨대 다음과 같이 서브 폰트 패밀리를 구성하면

모든 문화권에서 A(유니코드 65) ~ z(유니코드 122) 범위는 RobotoItalic 폰트를 사용하도록 강제할 수 있다.
만약 게임이 특정 문화일 때에 한해 적용하고 싶다면 위의 Cultures에 문화 코드(en, ko 등)를 입력하면 된다.

에셋 현지화

에셋의 현지화(예컨대 표지판 텍스쳐에 써있는 언어를 바꾼다던지)도 매우 쉽게 구현할 수 있다.
에셋을 우클릭하고 Asset Localization -> Create Localized Asset을 누르면 Content/L10N/문화코드 경로 밑으로 해당 에셋이 복사되는데, 이게 끝이다.

언리얼 게임은 에셋을 참조할 때 L10N/현재 게임의 문화코드 밑에 같은 경로와 이름의 에셋이 존재하면 그 에셋을 사용한다.

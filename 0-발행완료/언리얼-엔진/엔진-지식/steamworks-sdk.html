SteamPipe Build Uploader

스팀에 출시한 게임의 빌드 파일을 쉽게 업로드하고 업데이트할 수 있게 도와주는 GUI 툴.

스팀웍스SDK를 설치하면 내장되어 있다. tools 폴더 안에 SteamPipeGUI.zip 압축파일을 풀고 실행하자.

  • AppID에는 발급받은 스팀 AppID를 입력
  • Build Description에는 현재 빌드의 설명을 적는다.
  • Add Depot은 Depot(게임 빌드 파일에 대한 정보)을 추가한다. Depot ID는 일반적으로 AppID + 1부터 시작한다. 웬만하면 하나만 있으면 된다. Depot의 Build Path에는 게임 빌드 파일이 있는 경로를 입력한다. 언리얼 게임의 경우 실행파일이 있는 경로를 입력하면 된다.
  • Steamworks SDK ContentBuilder Path에는 스팀SDK경로\\tools\\ContentBuilder 를 지정한다.
  • Steam Login/Password에는 SteamWorks 아이디와 비밀번호를 적는다.

이후 Generate VDFs 버튼을 누르면 이 정보가 들어있는 vdf 파일들이 ContentBuilder/scripts 경로에 생성된다. (원래 들어있던 1000, 1001 얘넨 샘플 파일)

Upload 버튼을 눌러 스팀 서버에 업로드하자.

이후 스팀웍스 관리 페이지(partner.steamgames.com - Apps & Packages)에서 해당 게임을 누르고 SteamPipe 메뉴를 보면 업로드 기록을 볼 수 있다. 여기서 ‘Set build live on branch’를 ‘default’로 바꿔줘야 해당 빌드가 실제로 사용되므로 주의.

이후의 과정은 따로 다루도록 한다. 대충 Installation 메뉴에서 설치 경로, 어떤 파일이 실행 파일인지 지정하는 등의 작업이 필요하다.

AdvancedSession Plugin

언리얼의 온라인 서브시스템을 사용해 세션을 생성하고, 찾고, 입장하고, 종료하는 모든 과정을 쉽게 래핑해둔 플러그인.

세션 생성, 탐색, 입장: https://unrealengine.tistory.com/157

AdvancedSession 플러그인을 사용해 스팀과 연동하는 방법을 알아보자.

플러그인 설치하기

위 주소에서 AdvancedSessions, AdvancedSteamSessions 두 플러그인을 설치한다. 스팀웍스SDK는 언리얼이 Online Subsystem으로 이미 래핑해서 제공하므로 따로 설치할 필요 없다.

스팀 온라인 서브시스템 사용하기

Config/DefaultEngine.ini 파일에 다음 내용을 추가한다.

[/Script/Engine.GameEngine]
+NetDriverDefinitions=(DefName="GameNetDriver",DriverClassName="OnlineSubsystemSteam.SteamNetDriver",DriverClassNameFallback="OnlineSubsystemUtils.IpNetDriver")

[OnlineSubsystem]
DefaultPlatformService=Steam

[OnlineSubsystemSteam]
bEnabled=true
SteamDevAppId=480

[/Script/OnlineSubsystemSteam.SteamNetDriver]
NetConnectionClassName="OnlineSubsystemSteam.SteamNetConnection"

각 항목의 의미는 다음과 같다.

  • NetDriverDefinitions는 UE에서 사용할 넷 드라이버(클라이언트마다 하나씩 가지고 있는, 통신 로직을 처리하는 로우레벨의 객체)를 정의하는 것이다.
  • OnlineSubsystem의 DefaultPlatformService는 스팀 온라인 서브시스템을 사용하라는 의미.
  • OnlineSubsystemSteam의 항목들은 스팀웍스SDK와 연동하기 위해 필요한 데이터이다. DevAppId 480은 개발자용 공용 아이디이며, 이후 출시할 때 고유 ID를 발급받아 사용해야 한다.
  • OnlineSubsystemSteam.SteamNetDriver의 NetConnectionClassName은 스팀 온라인 서브시스템이 사용할 넷 드라이버 클래스의 이름이다.

세션 생성하기

AdvancedSessions 플러그인의 CreateAdvancedSession 노드를 사용한다.

세션을 성공적으로 생성했다면 최초 레벨을 열어 진입하며, Option에 ‘listen’을 명시하여 리슨서버 형태로 동작하도록 한다.

세션 찾기

세션 탐색에는 FindSessionAdvanced 노드를 사용한다. Results에 찾은 모든 세션의 정보가 담겨 전달된다. 대충 서버 브라우저 UI를 만들어서 목록화하면 된다.

세션 입장하기

JoinSession 노드를 사용한다. FindSessionAdvanced 노드를 통해 찾은 BlueprintSessionResult 객체를 그대로 사용하면 된다.

온라인 서브시스템 활용하기

모든 실습은 SteamWorksSDK 기준이다.

가장 먼저 OnlineSubsystem 플러그인을 활성화한다.

스팀SDK 연결하기

DefaultEngine.ini에서 다음 내용을 추가 혹은 수정한다.

[OnlineSubsystem]
DefaultPlatformService=Steam

[OnlineSubsystemSteam]
bEnabled=true
SteamDevAppId=480 // 개발용 공용 ID. 이후 출시할 때고유 ID를 발급받아 바꿔야 한다.

업적 추가/달성

https://www.youtube.com/watch?v=V0EsQaT7z14

게임의 업적 달성 기능은 대부분의 플랫폼 SDK에서 공통적으로 지원하며, 온라인 서브시스템을 사용해 아주 쉽게 구현할 수 있다.

스팀의 상품 페이지에서 Stats & Achievements - Achievements로 들어간다.

새 업적을 추가하고 알맞은 값을 집어넣는다.

API Name에 이 업적의 ID,

Display Name & Description에 달성했을 때 스팀이 띄워줄 팝업 UI에 출력될 업적의 이름과 설명,

Archieved/Unachieved Icon에 해금됐을 때 / 안 됐을 때 사용할 이미지를 넣고 저장한다.

이미 이것만으로도 API를 스팀웍스SDK로 연결시켜놓았을 경우 스팀 오버레이의 업적창에 추가한 업적이 뜨는 것을 볼 수 있다.

다음으로 DefaultEngine.ini에 업적 아이디를 등록한다.

[OnlineSubsystemSteam]
...
Archivement_0_Id=OpenDoor_ACH // 위에서 썼던 이름

이제 온라인 서브시스템 노드를 사용해 간단하게 업적 달성을 처리할 수 있다.

  1. Cache Achievements 노드로 스팀 서버(온라인 서비스)에서 플레이어의 업적 정보를 가져와 로컬 메모리에 캐싱한다.
  2. Get Cached Achievement Progress 노드로 캐싱된 업적 데이터에서 특정 ID의 Progress를 가져온다. Progress는 기본적으로 1 이상이면 달성이다. ‘몬스터 10마리 잡기’같은 건 0.1f로 표현하는 듯.
  3. Write Achievement Progress 노드로 스팀 서버로 변경된 진행도를 전달한다. 이 때 전달된 컨트롤러를 조종하고 있는 클라이언트의 스팀 업적이 달성된다.

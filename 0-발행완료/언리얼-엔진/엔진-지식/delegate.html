Delegate

개념

언리얼의 이벤트 시스템에 사용되는, Type-safe한 호출 대리자 객체.

언리얼 리플렉션 시스템과 밀접하게 통합되어 있어 UFUNCTION 매크로가 표기되어 리플렉션 시스템에 등록된 함수는 델리게이트에 간편하게 등록할 수 있다.

이를 통해 delegate에 바인딩된 함수 정보를 쉽게 저장하고 직렬화할 수 있으며, delegate가 참조하는 오브젝트가 소멸되었는지 여부를 IsBound()나 ExecuteIfBound() 함수로 안전하게 알고 처리하는 것도 가능하다.

델리게이트 객체 정의하기

델리게이트 타입은 언리얼이 제공하는 매크로로 정의한다. 다음은 자주 사용되는 매크로 목록이다.

// 기본 형태: DECLARE + DELEGATE + 반환 + 인자
// 이하 싱글캐스트 (단일 함수 바인딩)
DECLARE_DELEGATE(이름) // 반환과 인자 없음.
DECLARE_DELEGATE_@@@Param(이름, 인자타입...) // 인자 있음.
DECLARE_DELEGATE_RetVal(반환타입, 이름) // 반환 있음.
DECLARE_DELEGATE_RetVal_@@@Param(반환타입, 이름, 인자타입...) // 반환과 인자 있음.

// 응용
// 멀티캐스트 (다중 함수 바인딩, 브로드캐스트 가능)
DECLARE_MULTICAST_DELEGATE...
// 블루프린트 바인딩 지원 (다이나믹 델리게이트)
DECLARE_DYNAMIC_DELEGATE...
// 두 개 조합
DECLARE_DYNAMIC_MULTICAST_DELEGATE...

사용례

// 헤더
// delegate가 받을 수 있는 서명의 함수를 정의한다. UFUNCTION() 매크로로 표기하여 언리얼 엔진에 delegate로 받을 수 있는 함수임을(리플렉션 대상임을) 알린다.
UFUNCTION()
void OnBulletOverlap(UPrimitiveComponent* OverlappedComponent, AActor* OtherActor, UPrimitiveComponent* OtherComp, int32 OtherBodyIndex, bool bFromSweep, const FHitResult& SweepResult);

// 정의부 - delegate 바인딩
boxCollider->OnComponentBeginOverlap.AddDynamic(this, &ABullet::OnBulletOverlap);

기본적인 이벤트와 연결된 delegate는 컴포넌트 내부에 모두 정의되어 있다. OnComponentBeginOverlap은 delegate의 매개변수 구조체이며, 연결할 함수를 AddDynamic() 헬퍼 함수로 델리게이트가 대리할 함수를 바인딩한다.

Note

AddDynamic() 함수는 __internal_AddDynamic 함수의 헬퍼 함수(delegate가 요구하는 함수명 인자를 자동으로 전달해준다)로 인텔리센스가 자동으로 잡지 못 할 수도 있다. Add와 AddUnique라는 함수도 노출되어 있는데 이는 생성된 FDelegate 객체를 전달해 등록하는 함수로 __internal_AddDynamic 내부에서 사용한다.

Cooking

쿠킹이란?

언리얼 엔진은 에셋을 자체 포맷으로 저장한다. 그러나 빌드를 할 때는 플랫폼에 맞는 포맷이나 최적화된 포맷으로 변환하는데, 이 프로세스를 쿠킹이라고 한다.
쿠킹은 프로젝트 패키징 과정 안에서 일어난다. 패키징 과정은 대체로 다음과 같이 진행된다.

  1. 쿠킹 : 에셋을 플랫폼별 형식으로 변환한다.
  2. 스테이징(Staging) : 쿠킹된 에셋과 실행파일 등 필요한 파일들을 임시 폴더로 복사한다.
  3. 패키징(Packaging) : 스테이징된 파일들을 최종 배포 가능한 형태로 묶는다. (윈도우면 exe, 안드로이드면 apk)

쿠킹 대상 선정하기

언리얼의 자동 선정 기준은 ‘프로젝트가 시작할 때 필요한 에셋과, 그로부터 참조되는 모든 에셋’을 쿠킹 대상으로 지정하는 것이다.

보다 구체적인 내용은 다음을 참고:

1. **패키징 설정에 명시된 맵 (가장 중요한 기준):**
    - 프로젝트 세팅 > 프로젝트 > 패키징 > 빌드할 패키지에 포함할 맵 목록 (List of maps to include in a packaged build)
    - **이 목록에 추가된 맵들이 쿠킹의 가장 기본적인 시작점**이 됩니다.
    - 만약 이 목록이 비어 있다면, 아래의 다른 기준들이 더 중요해집니다. 하지만 **대부분의 프로젝트에서는 여기에 최소 하나 이상의 맵(보통 시작 맵)을 지정합니다.**
2. **기본 맵 (Default Maps):**
    - 프로젝트 세팅 > 맵 & 모드 (Maps & Modes) > 기본 맵 (Default Maps)
        - **게임 기본 맵 (Game Default Map):** 패키지된 게임이 실행될 때 가장 먼저 로드되는 맵입니다. 이 맵과 여기에 배치된 모든 액터, 그리고 그 액터들이 사용하는 에셋(머티리얼, 스태틱 메시, 블루프린트 등)들이 쿠킹 대상이 됩니다.
        - 에디터 시작 맵 (Editor Startup Map): 에디터를 열 때 기본으로 로드되는 맵. 패키징 시에는 '게임 기본 맵'이 더 중요합니다.
    - 만약 "빌드할 패키지에 포함할 맵 목록"이 비어있고, "게임 기본 맵"이 설정되어 있다면, 이 맵이 주요 쿠킹 대상이 됩니다.
3. **참조(Dependency) 관계:**
    - 위 1번 또는 2번에서 선정된 맵(들)에 포함된 모든 에셋(스태틱 메시, 스켈레탈 메시, 텍스처, 머티리얼, 블루프린트, 사운드 등)이 쿠킹 대상입니다.
    - 뿐만 아니라, 해당 에셋들이 **직접적이든 간접적이든 참조하고 있는 다른 모든 에셋들**도 함께 쿠킹됩니다.
        - 예: 맵 A에 스태틱 메시 B가 배치되어 있고, 이 스태틱 메시 B가 머티리얼 C를 사용하며, 머티리얼 C가 텍스처 D와 E를 사용한다면, 맵 A, 스태틱 메시 B, 머티리얼 C, 텍스처 D, 텍스처 E가 모두 쿠킹됩니다.
4. **프로젝트 설정에서 참조되는 에셋:**
    - 프로젝트 세팅 > 맵 & 모드 에서 설정된 기본 게임모드(Default GameMode), 기본 폰 클래스(Default Pawn Class), 플레이어 컨트롤러 클래스(Player Controller Class), HUD 클래스 등도 쿠킹 대상이 되며, 이 클래스들이 참조하는 에셋들도 포함됩니다.
    - 프로젝트 세팅 > 엔진 > 일반 세팅 (General Settings) 등에서 지정된 기본 에셋들도 포함될 수 있습니다.
5. **항상 쿠킹되는 콘텐츠 (Always Cook Content):**
    - 특정 폴더 (예: /Game/AlwaysCook)를 지정하여 해당 폴더 내의 모든 콘텐츠를 항상 쿠킹하도록 설정할 수 있습니다. (프로젝트 세팅에서 지정 가능)
    - 플러그인에 포함된 콘텐츠 중 항상 쿠킹되도록 설정된 콘텐츠도 포함됩니다.

따라서 아무것도 건드리지 않아도, 프로젝트가 실행 가능한 최소한의 에셋들은 엔진의 기본 설정과 참조 관계를 따라 쿠킹되지만, **가장 좋은 방법은 "빌드할 패키지에 포함할 맵 목록"에 실제 게임에서 사용될 맵들을 명시적으로 추가하여 쿠킹 대상을 정확히 지정하는 것**입니다. 이렇게 하면 불필요한 에셋이 쿠킹되는 것을 방지하고 빌드 크기를 최적화할 수 있습니다.

최적화

언리얼은 쿠킹 최적화를 위한 다양한 옵션을 제공한다.
대부분 Edit - Project Settings - Packaging - Packaging 섹션 - Advanced 섹션 안에 있다.

쿠킹된 컨텐츠 압축

Create compressed cooked packages 옵션을 켠다.
기본적으로 켜져 있다. 특히 모바일에서 중요하다. 많으면 APK 파일 용량을 50% 가까이 줄여준다고 함.

공유 머터리얼 셰이더 사용하기

패키지 크기는 줄일 수 있는데 로딩 시간이 좀 길어진다. 원하면 쓰자.

  • Share Material Shader Code (머티리얼 셰이더 코드 공유)
  • Share Material Native Libraries (머티리얼 네이티브 라이브러리 공유)

에디터 컨텐츠는 쿠킹에서 제외하기

에디터가 기본 제공하는 컨텐츠를 쿠킹에서 제외한다. 물론 이 옵션을 켜고 에디터가 제공하는 컨텐츠를 인게임에서 사용하면 문제가 생길 수 있으므로 조심하자.
Exclude editor content when cooking 옵션을 켠다.

쿠킹할 레벨 구체적으로 지정하기

Cook everything in the project content directory(ignore list of maps below) 옵션을 꺼서 컨텐츠 디렉토리의 모든 에셋 쿠킹을 끄고,
그 밑의 List of maps to inclue in a packaged build 배열에 쿠킹에 포함할 레벨을 구체적으로 지정할 수 있다. 옆에 … 버튼을 눌러 경로로 지정한다.

이러기 싫으면 그냥 다 꺼놓고 자동으로 맡기던가, 아님 Cook everything in the project content directory 플래그를 켜고, 그 바로 밑의 Cook only maps (This only affects cookall) 플래그도 켜면 게임의 모든 레벨에서 ‘실제로 사용하는’ 에셋만 쿠킹된다.

단, 이 옵션은 동적으로 로드되는 에셋들은 쿠킹 대상에서 제외될 수 있으므로 사용에 주의해야 한다.

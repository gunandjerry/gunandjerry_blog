UI(UMG)

C++로 UMG 다루기

UMG 기능은 블루프린트용 모듈로 따로 분리되어있어 Cpp에서 사용하려면 먼저 UMG 모듈을 등록해야 한다. 먼저 비쥬얼 스튜디오에서 [프로젝트명].build.cs 파일을 열면 PublicDependencyModuleNames 항목이 있다. 외부 모듈 파일을 프로젝트에 포함시키는 부분이다. 여기에 “UMG”를 추가하자.

PublicDependencyModuleNames.AddRange(new string[] { "Core", "CoreUObject", "Engine", "InputCore", "EnhancedInput", "UMG" });

UMG 오브젝트들은 모두 개별 객체로 생성된다. 다음은 두 개의 텍스트 박스를 생성하는 예시이다.

// 헤더
UPROPERTY(EditAnywhere, meta = (BindWidget))
TObjectPtr<class UTextBlock> currentScoreText;
UPROPERTY(EditAnywhere, meta = (BindWidget))
TObjectPtr<class UTextBlock> highScoreText;

UPROPERTY 내부의 ‘meta’ 라는 옵션은 메타데이터 지정자로, 코드를 통해 언리얼 에디터에 대한 제어를 할 때 사용한다. 예컨대 에디터에 노출되는 변수의 이름을 달리 하거나, 에디터에서 조절할 수 있는 입력 값의 범위를 제한하거나 할 수 있다.

여기서 사용된 BindWidget 옵션은 C++코드에서 만든 위젯 관련 변수를 실제 위젯 블루프린트와 연동시키는 역할을 한다.

해당 클래스를 상속하는 UserWidget 블루프린트를 상속하면 Bind Widget 탭에 BindWidget 메타 옵션을 지정한 포인터 목록이 나온다. 이제 Palette에서 객체를 배치한 뒤 이름을 Bind Widget에 있는 이름과 똑같게 설정하면 자동으로 바인딩이 된다.

Delegate 바인딩하기

UserWidget 클래스는 액터에서 파생된 것이 아니라 BeginPlay()가 없다. 대신 유사하게 동작하는 NativeConstruct()를 사용한다.

...
protected:
	virtual void NativeConstruct() override
	{
		Super::NativeConstruct();
		
		button_Restart->OnClicked.AddDynamic(this, &UMenuWidget::Restart);
	}

이 함수는 위젯이 생성되거나 초기화될 때 호출된다. 주의할 것은 위젯 컴포넌트에 SetWidget()을 하는 등 위젯을 할당하는 동작도 위젯의 초기화이므로 해당 함수를 호출한다는 것이다.

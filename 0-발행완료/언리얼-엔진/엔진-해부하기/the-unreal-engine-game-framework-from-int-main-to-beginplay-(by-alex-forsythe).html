언리얼 엔진의 루프 구조

게임 엔진의 일반적인 루프는 다음과 같다. 엔진을 초기화하고, 입력을 받고, 틱을 진행하고, 그려낸다.

int main()
{
	init();
	while (!g_exit_requested)
	{
		poll_input();
		update();
		render();
	}
	shutdown();
}

GuardedMain & FEngineLoop

이러한 구조는 언리얼 엔진도 마찬가지로, 핵심 코드만 남겨 언리얼 엔진의 메인 함수를 정리해보면 다음과 같은 형태가 된다.

int32 GuardedMain( const TCHAR* CmdLine )
{
	// Run early initialization, load engine modules
	int32 ErrorLevel = EnginePreInit( CmdLine );
	if ( ErrorLevel != 0 || IsEngineExitRequested() )
	{
		return ErrorLevel;
	}

	// Create and initialize a UEngine, run late initialization, start the game
	ErrorLevel = GEngineLoop.Init();

	// Every frame: kick off rendering, tick the engine, update RHI
	while(!GIsRequestingExit)
	{
		GEngineLoop.Tick();
	}
	
	// Run cleanup and exit when requested
	GEngineLoop.Exit();
	
	return ErrorLevel;
}

GEngineLoop.PreInit(CmdLine);

  1. .uproject 또는 .uplugin 파일에서 추가한 모듈들이 로드된다.
    • 엔진은 수 많은 모듈들의 조합으로 이루어지며, 어떤 모듈들은 핵심적인 기능을 한다. (Logging, Delegates, 3DMathLibrary, Reflection, GarbageCollection, InputKeyDefinitions 등)
    • 어떤 모듈들은 특정 플랫폼에서만 로드된다.
    • 모듈이 어떤 시점에 로드될 지 지정(ELoadingPhase)할 수 있다.
  2. CDO를 생성한다.

GEngineLoop.Init();

int32 FEngineLoop::Init()
{
	// Load the UGameEngine class that's specified in the Engine config file
	FString GameEngineClassName;
	GConfig->GetString(TEXT("/Script/Engine.Engine"), TEXT("GameEngine"), GameEngineClassName, GEngineIni);
	EngineClass = StaticLoadClass( UGameEngine::StaticClass(), nullptr, *GameEngineClassName);

	// Create a new UGameEngine and enshrine it as the global UEngine object
	GEngine = NewObject<UEngine>(GetTransientPackage(), EngineClass);
	check(GEngine);
	
	// Initialize the engine: this creates UGameInstance and UGameViewportClient
	GEngine->ParseCommandline();
	GEngine->Init(this);
	FCoreDelegates::OnPostEngineInit.Broadcast();
	
	// Initialize any late-loaded modules
	IProjectManager::Get().LoadModulesForProject(ELoadingPhase::PostEngineInit);
	IPluginManager::Get().LoadModulesForEnabledPlugins(ELoadingPhase::PostEngineInit);
	
	// Start the game: typically this loads the default map
	GEngine->Start();
	GIsRunning = true;
	FCoreDelegates::OnFEngineLoopInitComplete.Broadcast();
	
	return 0;
}
  1. 엔진 코어 클래스를 생성한다.
    • 새로운 레벨을 열거나 멀티플레이 서버에 접속하는 행위를 모두 Travel이라고 부르며, 이 때 그 목적지를 나타내는 문자열은 URL이 된다.
    • 로컬에서 레벨을 이동할 땐 레벨 에셋이 있는 경로가 URL이 되고, 멀티플레이 서버에 접속할 땐 서버의 IP 주소가 URL이 된다.
    • WWW에서 주소 뒤에 ?를 붙여 검색어를 전달하듯, 언리얼의 URL도 ?뒤에 여러 메타옵션을 담아 전달하게 된다. (ex. /Game/Maps/Stage1?listen -> Stage1 레벨을 리슨서버 호스트 모드로 열라는 뜻.)

UGameEngine::Init(IEngineLoop* InEngineLoop)

엔진은 맵이 로드되기 전에 자기 자신을 초기화하며, 이 때 중요한 몇 가지 오브젝트들을 생성한다.
GameInstance, GameViewportClient, LocalPlayer이다.

void UEngine::Init(IEngineLoop* InEngineLoop)
{
	// Create a UGameInstance using the class specified by project config
	FSoftClassPath GameInstanceClassName = GetDefault<UGameMapSettings>()->GameInstanceClass;
	UClass* GameInstanceClass = LoadObject<UClass>(nullptr, *GameInstanceClassName.ToString());
	GameInstance = NewObject<UGameInstance>(this, GameInstanceClass);
	
	// Create FWorldContext and dummy UWorld; initialize subsystems
	GameInstance->InitializeStandalone();
	
	// Create a UGameViewportClient and initialize it
	UGameViewportClient* ViewportClient = NewObject<UGameViewportClient>(this, GameViewportClientClass);
	ViewportClient->Init(*GameInstance->GetWorldContext(), GameInstance);
	GameViewport = ViewportClient;
	GameInstance->GetWorldContext()->GameViewport = ViewportClient;
	CreateGameViewport(ViewportClient);
	
	// Create a ULocalPlayer and associate it with the viewport client
	FString Error;
	ViewportClient->SetupInitialLocalPlayer(Error);
	UGameViewportClient::OnViewportCreated().Broadcast();
	
	// Done; now we're ready to load PostEngineInit modules and start the game
	UE_LOG(LogInit, Display, TEXT("Game Engine Initialized."));
	bIsInitialized = true;
}
  • UGameInstance는 게임의 전역적인 관리자로, 언리얼 4.4에서 UGameEngine 클래스 중 게임 프로젝트와 밀접하게 관련이 있는 일부 함수를 들고 떨어져나왔다. 게임을 플레이하는 내내 살아있는 최상단의 전역 관리자로 여러 데이터를 저장하는 용도로도 사용된다.
  • UGameViewportClient는 렌더링, 오디오, 인풋 시스템과 게임 사이의 최상단 인터페이스이다. 로컬 플레이어를 위한 뷰포트를 관리하며 사용자의 입력을 가장 먼저 처리하기도 한다.
  • ULocalPlayer는 로컬 환경상의 플레이어를 나타낸다. 요즘은 컴퓨터 한 대에 컨트롤러 두 대를 연결하는 일은 잘 없으니 대개 한 명이다. (네트워크를 통해 연결된 리모트 플레이어는 UNetConnetion으로 나타낸다.)

UGameEngine::Start()

UGameEngine, UGameInstance, UGameViewportClient, ULocalPlayer가 준비되었다면, 엔진이 작동하기 시작한다.
여기서 레벨을 로드하게 된다. 콜스택을 살펴보면 다음과 같다.
FEngineLoop::Init -> UGameEngine::Start -> UGameInstance::StartGameInstance -> UEngine::Browse -> UEngine::LoadMap

UEngine::LoadMap에 성공해 레벨이 로드되면 다음과 같은 계층구조를 가지는 월드 객체를 가지게 된다.

  • UWorld : 단일 월드를 나타냄.
    • ULevel : 월드에 존재하는 액터들을 관리하는 주체. UWorld가 직접 하지 않는 이유는 언리얼은 StreamingLevel을 통해 여러 레벨을 하나의 월드에 로드하고, 각각의 레벨들이 관리하는 액터들을 구분할 수 있기 때문이다.
      • AActors : ULevel에 의해 관리되는 개별 액터들.
        • UActorComponent : 액터에 부착되는 컴포넌트들.

AGameModeBase, AGameStateBase, APawn 등 모든 액터들은 월드가 생성된 뒤 스폰된다. 따라서 언리얼의 UObject는 두 가지 성격으로 분리할 수 있는데,

하나는 UGameEngine, UGameInstance, UGameViewportClient, ULocalPlayer와 같이 맵이 로드되기 전, 엔진 초기화 단계에서 생성되는 객체로, 이들을 Engine Objects라고 부른다.
또 하나는 UWorld, AActor와 같이 엔진이 시작되고 맵이 로드된 후 생성되는 객체로, 이들을 Game Objects라고 부른다.

Game Objects들은 어디까지나 현재 맵 위에서 생성된 객체들이므로, 맵이 전환되면 기존의 게임 오브젝트들은 모두 제거된다. 반면 엔진 오브젝트들은 레벨과 무관하게 존재하므로 제거되지 않는다.

Seamless Travel에서 액터의 보존

Seamless Travel 방식은 레벨 전환 과정에서 액터의 Outer(Owner)를 변경할 수 있어 액터를 보존할 수 있다. 자세한 건 문서 참고

UEngine::LoadMap

새 레벨을 로드한다. 이 과정은 엔진 초기화 과정에서 UGameInstance에 의해 생성되는 월드 컨텍스트 객체에 의해 추적되고, 현재 월드가 어떤 월드인지 자동으로 갱신된다.
맵을 로드하는 과정을 핵심적인 부분만 살펴보면 다음과 같다.

  1. 새 맵을 로드하기 전, FCoreUObjectDelegates::PreLoadMap 델리게이트를 브로드캐스팅한다.
  2. 기존 맵이 있다면 해당 맵 위에서 생성된 모든 게임 오브젝트를 제거한다(UWorld도 클린업됨).
  3. 필요하다면 게임 인스턴스는 레벨 로드 전에 필요한 에셋을 먼저 로딩할 수 있다(WorldContext.OwningGameInstance->PrloadContentForURL(URL);). 다만 대개는 아무것도 하지 않는다.
  4. 맵 패키지(월드 패키지)를 로드한다. 일반적으로 .umap 확장자로 저장되는 직렬화된 파일로, 내부엔 UWorld, ULevel, AActors, UActorComponents 들이 직렬화되어 저장되어 있다.
    이 단계에서 각 개별 객체들의 초기화 단계(변수가 어디까지 초기화, 또는 함수가 어디까지 호출되었는가)를 살펴보면 아래와 같다.
  5. 로드된 새 월드(UWorld)에 OwningGameInstance를 설정한다. 이제 월드를 통해 게임 인스턴스에 접근할 수 있다.
  6. 로드된 새 월드를 GC의 루트 객체로 지정한다.
  7. 물리, 네비게이션, AI, 오디오 등 여러 시스템을 초기화한다.
  8. GameInstance에게 요청하여 게임 모드 액터를 생성하게 한다. (SeamlessTravel로 보존되었다면 생략)
  9. 패키지에서 'always loaded'로 설정된 서브 레벨들을 모두 로드한다.
  10. UWorld::InitializeActorsForPlay를 호출하여 액터에 필요한 변수를 초기화하고, 일련의 초기화 절차를 거친다.
    • bActorsInitialized 플래그를 켜서 이후로는 액터가 스폰될 때 초기화되도록 한다.
    • 액터의 컴포넌트들을 초기화한다. 이 때 bAutoActivate 플래그가 켜져있다면 바로 활성화(Activate(true))되며, bWantsInitializeComponent 플래그가 켜져있다면 InitializeComponent()를 호출한다.

이제 게임과 상태를 관리하는 주요 액터들(AGameModeBase, AGameSession, AGameNetworkManager, AGameStateBase)을 보유하고 있으며, 게임을 시작할 준비를 마쳤다.

  • AGameModeBase: Server-only. 게임의 룰을 정의하며, 대부분의 코어 액터들을 스폰한다.
  • AGameSession: Server-only. 멀티플레이 게임에서 로그인 요청 등을 처리하며, 온라인 서비스(스팀, 에픽, PSN 등)와 게임의 인터페이스로서도 사용된다.
  • AGameNetworkManager: Server-only. 이동 보간, 치트 감지 등.
  • AGameStateBase: Server-Authoritative. 모든 게임 인스턴스에 생성되지만 서버만 수정권한을 가지며, 클라로 리플리케이트된다. 모든 플레이어들이 알아야 하는 데이터 관리에 사용.
  1. 모든 LocalPlayer를 순회하며 ULocalPlayer::SpawnPlayActor() -> UWorld::SpawnPlayActor()를 통해 플레이어 컨트롤러를 생성한다. PlayActor는 플레이어 컨트롤러라고 이해하면 된다.
Note

ULocalPlayer는 UEngine::Init에서 생성되는 엔진 오브젝트로, 엔진에서 접속한 플레이어를 나타낸다.
반면 APlayerController는 UEngine::LoadMap 과정에서 생성되는 게임 오브젝트로, 실제 게임, 월드에서 개별 플레이어를 나타낸다.

UWorld::SpawnPlayActor()

중요한 함수라 잠깐 짚고 넘어가자. PlayActor는 그냥 플레이어 컨트롤러라고 생각하면 된다. 이 함수에서 플레이어의 접속 절차를 처리하게 된다.
로컬 플레이어(ULocalPlayer)든 리모트 플레이어(UNetConnection)든 로그인 절차를 거친 뒤 문제가 없어야 컨트롤러를 생성할 수 있으며, 이 과정은 게임 모드가 담당한다.

  1. AGameModeBase::PreLogin()은 리모트 플레이어가 접속을 시도할 때 로그인 요청을 수락할 지 말지 결정한다. 수락되면 Login으로 넘어간다.
  2. AGameModeBase::Login()에서 플레이어 컨트롤러 액터를 월드에 생성한다.
    • APlayerController::PostInitializeComponents에서 플레이어 스테이트도 생성. 플레이어 스테이트는 게임 스테이트처럼 모든 클라이언트에게 전파되며, 해당 플레이어에 관해 모두가 알아야 하는 데이터를 관리한다.
    • 네트워크 Role, 연결된 UPlayer 등을 설정한다.
  3. AGameModeBase::PostLogin()에서 만들어진 플레이어 컨트롤러가 빙의할 새 폰을 생성한다.
    • bStartPlayersAsSpectators 플래그가 켜져있다면 월드를 날아다닐 수 있는 기본 스펙테이터 폰을 생성한다.
    • 그렇지 않다면 RestartPlayer()를 호출하여 PlayerStart가 있는 위치에 새 폰을 생성한 후, AController::SetPawn()으로 빙의시킨다. 이 때 APlayerController::OnPossess와 APawn::Possessed가 호출됨.
UNetConnection

ULocalPlayer는 UPlayer를 상속하여 로컬 플레이어를 나타내는데, 네트워크를 통해 연결된 리모트 플레이어는 ULocalPlayer가 아닌 UNetConnection 객체로 관리된다. 이 클래스 역시 UPlayer를 상속한다.

UWorld::BeginPlay()

월드를 시작하는데 필요한 모든 준비물(월드, 액터들, 게임과 액터를 제어할 필수 액터들)이 갖춰졌다.
이제 엔진 -> 월드 -> 게임모드 -> 월드세팅의 순서로 BeginPlay를 처리하도록 명령하며, 월드 세팅에서 모든 액터를 순회하며 BeginPlay()를 호출하게 된다.
또, 액터의 BeginPlay()에서 자신의 모든 컴포넌트에 대해 BeginPlay()를 호출한다.

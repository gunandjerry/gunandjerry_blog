메타휴먼을 플레이어 캐릭터로 사용하기

Note

언리얼 샘플 3인칭 프로젝트의 마네킹 메쉬는 두 종류(UE4 마네킹과 Manny&Quinn)인데, 전자는 언리얼4까지 쓰던 메쉬이고 후자는 언리얼5에서 추가된 본이 더 많은 버전이다.
언리얼 5의 특정 버전부터 메타 휴먼 스켈레톤에 Assgin Skeleton으로 본 머지를 시도하는 작업이 UE4 버전 마네킹으론 안 되게 되었다. Manny 또는 Quinn을 기준으로 작업하자.

플레이어 메쉬로 사용하기

방식1. ChildActor로 부착

메타휴먼 캐릭터를 Quixel Bridge에서 임포트하면 생성되는 BP를 통채로 CharacterMesh(이하 '기본 메쉬')의 ChildActor로 부착하고,
Live Retargeting을 통해 메타휴먼 메쉬가 기본 메쉬의 본과 포즈를 따라가게 해보자.

기본 메쉬에 애니메이션 블루프린트를 부착하되 메쉬는 넣지 않고, Optimization 섹션의 Visibility Based Anim Tick Option을 Always Tick Pose and Refresh Bones로 바꾼다.
이후 ChildActor의 Child Actor Component - Child Actor Template - Live Retarget - Use Live Retarget Mode를 체크한다.

이러면 실시간으로 부모(아마도 LeaderPose로 설정된) 스켈레탈 메쉬의 본을 리타게팅해서 따라가게 된다. 만약 안 된다면 기본 메타휴먼 리타겟 애님 클래스가 누락된 것이니 Child Actor Template - Live Retarget - Retarget Anim Class에 _Retargeting 으로 끝나는, 함께 임포트된 메타휴먼 리타게팅 ABP를 부착하도록 한다.

장점
  • 쉽다.
단점
  • 이유는 모르겠으나 매번 Retarget Anim Instance가 누락되고 다시 끼워줘야 라이브 리타게팅이 정상 작동했다. 레딧의 모 유저는 Construction Script에서 매번 시작할 때마다 끼우라는데, 이러면 어떤 MetaHuman 액터를 쓸 지 미리 정해놔야 하지 않는가?

스켈레탈 메쉬 직접 사용하기

메타휴먼 캐릭터는 대략 최상위 메쉬가 되는 Body와 그 밑의 Face, Torso, Legs, Feet 메쉬들로 이루어진다. 이 중 Face는 별도의 메타휴먼 페이스 스켈레톤을, 나머지는 메타휴먼 스켈레톤을 사용한다.

이 스켈레탈 메쉬들을 직접 사용해도 된다. 구조는 메타휴먼 캐릭터 BP에 있는 그대로 바디를 기준으로 자식 스켈레탈 메쉬들을 부착하면 된다.
애니메이션들이 모두 기본 마네킹을 사용하거나 그런 경우 필요하다면 각 스켈레탈 메쉬들에 대해 우클릭 -> Skeleton -> Assign Skeleton을 통해 스켈레톤을 변경할 수 있다.
경험상 UE5의 기본 스켈레톤은 잘 호환이 된다. (손가락 관절은 많이 날라간다.)

Note

기본적으로 자식 스켈레탈 메쉬 컴포넌트는 부모 스켈레탈 메쉬 컴포넌트를 LeaderPose로 지정하고 따라간다. 그런데 종종 일부 컴포넌트에서 이 설정이 자동으로 이루어지지 않는 경우가 있다. 이 때는 수동으로 SetLeaderPoseComponent를 호출해줘야 한다.
(당연하지만 자식 스켈레탈 메쉬에 같은 ABP를 지정할 필요는 없다.)

동적으로 스켈레탈 메쉬 변경하기

메타휴먼 BP를 받아 동적으로 스켈레탈 메쉬를 교체하는 기능을 만들어보자.

먼저 스켈레탈 메쉬의 하이어라키는 다음과 같다. 스켈레탈 메쉬 컴포넌트들의 경우 그냥 스켈레탈 메쉬 에셋만 얻어서 갈아끼워주면 된다.
다만 Face Mesh 아래에 부착되는 UGroomComponent 들의 경우 설정이 제각각이고 가져와야 하는 에셋도 많아 그냥 복사 생성했다.

메타휴먼 BP에는 두 가지 액터 컴포넌트(MetaHuman, LODSync)가 옵션으로 따라온다.

  • MetaHuman Component : 메타휴먼 스켈레탈 메쉬의 각 LOD 레벨별로 바디 보정, 페이셜 애니메이션 처리, 의상 물리 처리 등을 담당한다고 한다. 정확하지 않다.
  • LODSync Component : 여러 LOD 레벨 간의 부드러운 전환을 처리하는 역할.
    LOD를 사용하지 않는다면 전자만 가져오면 된다고 한다.

LODSync 컴포넌트의 경우 요상한 버그가 발생한다. 다음 문서 참고.
메타휴먼 LOD 버그

블루프린트 컴포넌트 순회

메타휴먼 BP를 전달하면 알아서 연관 컴포넌트를 찾아 에셋을 빼와야 하는데, 액터 인스턴스가 아니라 블루프린트 에셋을 그대로 전달해야 하므로 GetComponent 류의 인스턴스에 대해 런타임에 동작하는 함수는 사용할 수 없다. 대신 CDO에 접근해야 하는데, 블루프린트의 경우 UBlueprintGeneratedClass로 캐스팅해서 노드를 순회하며 이름으로 찾는 것이 가장 빠르다.

void AFHPlayerBase::ChangeMetahuman(TSubclassOf<AActor> MetahumanActor)
{
	if (MetahumanActor == nullptr) return;

	TArray<TPair<USCS_Node*, FString>> MetaHumanSkeletalMeshes;
	TArray<TPair<USCS_Node*, FString>> MetaHumanGrooms;

	UBlueprintGeneratedClass* BPClass = Cast<UBlueprintGeneratedClass>(MetahumanActor);
	if (USimpleConstructionScript* SCS = BPClass->SimpleConstructionScript)
	{
		for (USCS_Node* Node : SCS->GetAllNodes())
		{
			if (Node && Node->ComponentTemplate)
			{
				USkeletalMeshComponent* SkeletalMesh = Cast<USkeletalMeshComponent>(Node->ComponentTemplate);
				if (SkeletalMesh)
				{
					MetaHumanSkeletalMeshes.Add({Node, SkeletalMesh->GetName()});
					continue;
				}

				UGroomComponent* GroomComp = Cast<UGroomComponent>(Node->ComponentTemplate);
				if (GroomComp)
				{
					MetaHumanGrooms.Add({Node, GroomComp->GetName()});
					continue;
				}
			}
		}
	}

	// 이후 이름으로 찾는다.
	// 블루프린트 노드의 컴포넌트 이름의 경우 "@@@_GEN_VARIABLE"의 형태이다.
	
	for (auto& pair : MetaHumanSkeletalMeshes)
	{
		FString name = pair.Value;
		USkeletalMeshComponent* SkeletalMesh = Cast<USkeletalMeshComponent>(pair.Key->ComponentTemplate);
		if (name == TEXT("Body") || name == TEXT("Body_GEN_VARIABLE"))
		{
			GetMesh()->SetSkeletalMesh(SkeletalMesh->GetSkeletalMeshAsset());
		}
		// (...생략...)
}

메타휴먼 래그돌

퀵셀 브릿지에서 메타휴먼 캐릭터를 임포트하면 바디 피직스 에셋까지 제공해준다.
기본 경로는 Content/MetaHumans/Common/{성별}/{키}/{체격}/Body
안에 보면 기본 상태일 때 애니메이션 적용하라고 주는 기본 피직스 바디와 래그돌 상태일 때를 가정하여 PhysicsConstraint의 각도가 제한되어 있는 ragdoll 버전 두 개를 제공한다. 래그돌이 되면 그걸로 갈아끼워야 한다.

capsuleComp->SetCollisionEnabled(ECollisionEnabled::NoCollision);

thirdPersonMesh->SetCollisionProfileName(TEXT("Ragdoll"));
thirdPersonMesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);
thirdPersonMesh->SetAllBodiesSimulatePhysics(true); // 피직스 바디 시뮬레이션 켜기
thirdPersonMesh->SetSimulatePhysics(true);
thirdPersonMesh->SetPhysicsAsset(PlayerDescriptor->RagdollPhysicsAsset);

thirdPersonMesh->WakeAllRigidBodies();

구글링을 해본 결과 스켈레탈 메쉬의 SetPhysicsAsset 동작은 SetSimulate 이전에 호출하는게 안전하다고 하는데, 메타휴먼의 경우 그 전에 호출하면 희한하게도 GroomComponent가 스켈레탈 메쉬를 빠져나가고, 메쉬는 땅이랑 겹치고 난리가 난다.
이후에 갈아끼우면 정상적으로 작동한다...

또, FaceMesh에 별도의 페이셜 애니메이션을 적용한 뒤 바디와 블렌딩하기 위해 BodyMesh를 FaceMesh의 Leader pose로 사용하는 대신, 페이스 애니메이션에서 Copy pose from skeletal mesh 노드로 바디 포즈를 복사하여 블렌딩하고 있는데,
이 상태에서 래그돌이 되면 애니메이션 포즈를 인식할 수 없기 때문에 FaceMesh와 BodyMesh가 어긋난다. 따라서 래그돌이 됐을 때는 임시적으로 BodyMesh를 LeaderPose로 설정해줘야 한다.

크리쳐한테 맞아서 날아가는 것도 만들어보았다.
